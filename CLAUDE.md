# Project: schedule-share

## 目標
- 誰でも簡単に予定を共有・管理できる、シンプルで汎用的なスケジューラーの構築。
- 清潔感があり、直感的に操作できるユーザーインターフェース。

## 開発ルール
- **技術選定**: Next.js (App Router), TypeScript, Tailwind CSS を採用する。
- **命名規則**: 汎用的な用語（Event, Schedule, Category）を使用する。特定の競技や特定の業務に特化した固有名詞は禁止。
- **コード規約**: 
  - コンポーネントは src/components/ に配置。
  - データ取得やDB連携は src/lib/ 内のモジュールに集約し、コンポーネント内でのハードコードを避ける。
  - インポートは @/ エイリアスを使用する。
- **UI デザイン**: 
  - ミニマルなデザインを追求する。
  - 特定のカテゴリにとらわれない、ニュートラルな配色を基本とする。
